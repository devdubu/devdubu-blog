# Cloud 컴퓨팅의 유형
## Public 클라우드
![Pasted image 20231023090937.png](../../../Pasted image 20231023090937.png)
- 사용자는 컴퓨팅 리소스를 <mark>소유하지 않는다.</mark>
- 인터넷을 통해 제공합니다.
- 가상화 기술로 만든 서비스를 그대로 사용합니다.

## Private 클라우드
![Pasted image 20231023091157.png](../../../Pasted image 20231023091157.png)
- 특정 조직 내에서 컴퓨팅 리소스를 <mark>소유</mark>
- Private 네트워크를 통해 제공
- 가상 컴퓨팅 기술을 직접 구축

|유형|설명|
|--|--|
|퍼블릭(public) 클라우드| - 사용자는 컴퓨팅 리소스를 <mark>소유하지 않는다.</mark> <br /> - 인터넷을 통해 제공합니다. |
| 프라이빗(private) 클라우드 | - 특정 조직내에서 컴퓨팅 리소스를 <mark>소유</mark> <br />- <mark>프라이빗 네트워크</mark>를 통해 제공 |
|하이브리드(hybrid) 클라우드| - 퍼블릭 클라우드와 프라이빗 클라우드 또는 데이터센터 간 <mark>네트워크 연결</mark> <br /> - 데이터 및 애플리케이션을 공유 |
|멀티(Multi) 클라우드|- <mark>다수의 퍼블릭 클라우드</mark>를 사용 <br /> - e.g.AWS+GCP+Azure|

# 클라우드 서비스의 특징
:::question 클라우드 서비스?
- 인터넷을 통해 IT 리소스와 애플리케이션을 사용자가 원할 때 언제든지(on-demand) 사용한 만큼 요금을 내는 서비스
![Pasted image 20231023092528.png](../../../Pasted image 20231023092528.png)

:::

:::tip 탄력성 / 민첩성
- 리소스에 대해 필요할 때 언제든 늘리고, 줄일 수 있다.

:::

:::bug 확장성
- 물리 서버를 확장하려면 시간이 오래 걸리는 반면, 클라우드는 즉시 확장이 가능

:::

:::note 사용한 만큼 지불
- 전기 요금 처럼 사용한 만큼 과금 되며, 비용 예측이 가능하다.

:::

:::success 내결함성 및 재해복구
- 클라우드 백업 및 클라우드 DR 구성으로 데이터 손상 등 긴급 상황에 대처 가능

:::

:::warning 고 가용성
- 손쉬운 다중 가용 영역 설정에 따라 고 가용성을 보장

:::

:::error 유지 관리 간소화
- 물리적인 리소스를 유지할 필요가 없고, 부분적인 클라우드 CSP 벤더에 위임


:::
