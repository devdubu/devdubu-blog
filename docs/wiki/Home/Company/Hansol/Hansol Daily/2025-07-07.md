# 문제점

nuxt 자체적으로 해당 type을 체크 할 때는 type의 어려운 추론 까지 무난하게 가능
즉, 디렉토리 구조로 extends를 받을 때는 그리 type에 대한 문제들이 나오지 않음

문제는 `node_module` 로 들어갈때 문제가 생김

실제 nuxt에서 나오는 type에 대한 내용은 `.nuxt` 해당 내용이 들어갈 뿐만 아니라, 해당 경로에도 많은 부분들이 엮여 있음
그렇기 때문에 ts 에서 현재 우리가 복잡한 추론을  제시를 해도 어느정도 추론이 가능함

하지만, `node_module`에 들어가는 순간 현재 nuxt에서 사용하고 있는 복잡한 추론 시스템을 사용하지 못함
그리고 문제는 nuxt layer로 사용하여 nuxt extends install을 한 부분은 완벽하게 npm build를 통해서 타입과 모든 컴포넌트들이 말려 있는 상태가 아니다보니, 
실제 복잡한 타입을 추론하는데 있어서 한계가 나오면서 type들이 꼬이는 현상들이 발생함

결국 디렉토리 extends 방식과 github install 방식의 차이이다.
