# Chained Select

## 1안

:::note
💡

기존 Select Box에서 부수적인 type을 추가해서 chain Select Box 구현

:::

### 고려 요소

- 특정 데이터를 선택 시에 다른 데이터를 고를 수 있게 해줄 것인지
    
    - 즉, select column 하나 마다 각기 다른 데이터를 넣을 것인지
- `"area1code":"11","area1name":"서울특별시","area2code":"11010","area2name":"종로구","area3code":"1101053","area3name":"사직동”`
    
- 위 처럼 select 데이터를 한꺼번에 받고, 앞 단에서 분기 처리를 할 것인지?
    
- 위 고려 사항 1가지만 고려, 혹은 둘 다 고려 하는 것에 따라서 grid는 로직이 바뀔 가능성이 큼
    

```tsx
{
	text: '셀렉트박스',
  field: 'DATA_CODE1',
  width: 150,
  editor: {
	  type: 'input-select',
    textField: 'CODE_GRP_NM',
    valueField: 'CODE_GRP_CD',
    api: {
		    url: '/api/admin/code/code-grp',
	  },
    data: [
	    {
	      'CODE_GRP_NM': '공통 코드 예시',
        'CODE_GRP_CD': 'SYS0001',
      },
    ],
  },
},
{
	text: '셀렉트박스',
  field: 'DATA_CODE2',
  width: 150,
  editor: {
	  type: 'input-chain-select',
		chainFields: ['DATA_CODE1', 'DATA_CODE2', 'DATA_CODE3'],
		chainTarget: 'DATA_CODE'
    textField: 'CODE_GRP_NM',
    valueField: 'CODE_GRP_CD',
  },
},
{
	text: '셀렉트박스',
  field: 'DATA_CODE3',
  width: 150,
  editor: {
	  type: 'input-chain-select',
		chainField: ['DATA_CODE1', 'DATA_CODE2', 'DATA_CODE3'],
		chainFilter: 'Func'
    textField: 'CODE_GRP_NM',
    valueField: 'CODE_GRP_CD',
    
  },
},

function Func (args)=>{
	
	// 로직
	
	
	// chain 에 필요한 리턴 데이터
	return { chainFilter1, chainFilter2 }
}
```

- `chainFilter` 를 함수로 제공한다면, 상단 셀렉트 박스에 api 로 받은 데이터를 함수의 매개변수로 전달할 필요가 생김
    - 해당 매개변수를 통해서 내부적으로 select 해서 처리 하게 구현해주면 될 듯 싶음
- level 1 의 셀렉트박스 에서는 기존처럼 사용 - 모든 api 데이터는 이곳에서
- chainField의 Array Index가 레벨로 고려하기
