## Store 에서 dialog / router tap 삭제

- 현재 기존에는 `store/functionStore` 에서는 set 이벤트 시 그냥 그냥 push 를 하는 형태이고,
- 실질적으로 같은 pcd가 여러개가 들어와있는 환경에서 재귀 함수를 돌았을 때, push 를 했던 index는 돌지 않는 것으로 확인 된다.
- 고려 할 요소는 우선, 창을 닫을 때 혹은 delete와 유사한 이벤트 시에, 해당되는 pcd 값을 추적해서 삭제할 수 있어야 한다.
- dialog 끼리는 중복을 고려 하지 않아도 된다.
- 대신 router tab 간의 중복이 있을 수도 있기에, dialog 끼리의 대조와 router tab간의 대조는 사실상 의미가 없다.
- 아 하긴, 애초에 컨셉이 HDialog나 Hcontainer나 모호해서 별로 차이는 없을 듯 하다
