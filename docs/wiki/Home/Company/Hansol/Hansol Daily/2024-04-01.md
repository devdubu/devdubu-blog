### typescript에서 modules에 관한 Path 에러가 나는 이유

```
일반 JavaScript에서는 모듈 경로 문제가 발생하지 않는 것은 맞습니다. 
그 이유는 JavaScript는 런타임에서 모듈을 동적으로 로드하기 때문에 모듈의 실제 위치를 찾아 사용할 수 있기 때문입니다. 
반면에 TypeScript는 정적인 타입 체크를 수행하고 컴파일 과정에서 모듈을 분석하기 때문에 모듈의 경로를 정확하게 알아야 합니다.

TypeScript는 컴파일 시점에 모듈을 찾고 타입 체크를 수행하기 때문에 정확한 모듈 경로를 지정해야 합니다. 따라서 TypeScript에서 모듈 경로 문제가 발생할 경우, 
이는 TypeScript 컴파일러가 해당 모듈을 찾을 수 없어서 발생하는 것입니다. 일반 JavaScript에서는 이러한 정적인 타입 체크가 이루어지지 않기 때문에 모듈 경로 문제가 나타나지 않는 것입니다.

따라서 TypeScript에서는 모듈을 찾을 수 있는 올바른 경로를 설정해야 합니다. 
이는 tsconfig.json 파일에서 paths 설정을 통해 수행할 수 있습니다. 
이 설정은 TypeScript에게 모듈의 실제 위치를 알려주어 정확한 타입 검사와 모듈 로드를 가능하게 합니다.
```

- 즉, `tsconfig.json` 파일에서 절대 경로를 지정해야한다.
[[Typescript] tsconfig 절대경로 설정하기](https://medium.com/@relee6203/typescript-tsconfig-%EC%A0%88%EB%8C%80%EA%B2%BD%EB%A1%9C-%EC%84%A4%EC%A0%95%ED%95%98%EA%B8%B0-4db1870767f8)
### realgrid 측의 `tsconfig.json`

```json
{
  "compilerOptions": {
    "target": "ESNext",
    "useDefineForClassFields": true,
    "module": "ESNext",
    "moduleResolution": "Node",
    "strict": true,
    "jsx": "preserve",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "esModuleInterop": true,
    "lib": ["ESNext", "DOM"],
    "skipLibCheck": true,
    "noEmit": true,
    "types": ["vite/client"]
  },
  "include": ["src/**/*.ts", "src/**/*.d.ts", "src/**/*.tsx", "src/**/*.vue"],
  "references": [{ "path": "./tsconfig.node.json" }]
}
```
