## Test

Grid에서의 테스트 진행

- 로직 테스트를 진행하고 있으며,
- return 값을 최소한의 값으로 뱉는 것을 테스트를 할 것인지, 정확히 이런 것을 반환하는 것을 할것인지에 대한 고민
    - 최소한의 값을 뱉는 것도 좋지만, 리턴 하는 옵션이 달라지면, 달라질수록 실제로 구현되는 것들이 달라지기에, 최소로 할 것인지, 아닌 지에 대한 고민은 지속적으로 필요 해보임
- 사실 상 틀린 값에 대한 Error를 뱉는 게 중요할 것 같다.
    - 올바른 값에 대한 부분은 그리 크게 문제가 되지 않지만, 여러가지 가짓수에 대해서, Error와 Warning을 뱉어주는 것이 올바른 로직 일 것 같다.

**Warning**

```tsx
throw new Warning("Something is wrong!");
```

**Error**

```tsx
throw new Error('Error')
```

그럼 여기서 고민 되는 부분은 Warning과 Error는 어떻게 분리를 할 것인가.

다음은 Error와 Warning의 차이입니다.

**Error**는 Compile error, Runtime error와 같이 문제를 교정하지 않으면 원활히 실행시킬 수 없는 상태를 말합니다.

**Warning**는 프로그램 실행은 가능하지만 잠재적인 문제를 가지고 있는 상태를 말합니다. 예를 들어, 선언한 변수를 사용하지 않았거나, Scanner 객체를 사용한 후 `System.in` 스트림을 close하지 않아서 `Resource Leak`가 발생하는 경우 등이 있습니다.
