# NButton

- 아이콘의 위치 자체를 CSS 에서 어떤 식으로 컨트롤 할지에 따라서 해당 태그 모양이 변경될 여지가 있음

현재는 span, image 태그가 같을 레벨로 되어있음
```vue
<template>
	<button
		:data-type="`HButton-${buttonName}`"
		type="button"
		:class="`nui_btn ${buttonClass} ${buttonHide}`"
		:style="`${buttonWidth} ${buttonStyle}`"
		:disabled="buttonDisabled"
		@click="handleClickButton">
	<img v-if="props.imageIcon" :src="props.imageIcon" />
	<span v-if="buttonIconCls" :class="buttonIconCls" />
	<slot name="icon" />
	<span v-if="buttonText">{{ buttonText }}</span>
	</button>
</template>
```

상황에 따라서 icon을 상위 개념을 묶을 수도 있음

- 해당 아이콘을 호출하려면 무조건 public에 이미지가 있어야함
- 기본적으로 nut-ui public 폴더를 공유하기 때문에, 이 부분 또한 extends 영향을 받는 곳임
	- asset으로 경로 하달 시, 뜨지 않음

현재는 slot 으로 인한 내부 이미지 배치, image url, icon class 태그 등 대응 함
playnut에서 mdi 뜨는 것은 확인 완료
