# HChart

## interface

type - 필수 - `sting`
title - 필수 - `string`
options - 옵션
subtitle - 옵션 - `string`
xAxis - 옵션
yAxis - 옵션
series - 옵션

### config

```tsx
interface HChartUi{
	type: string | null
	title: string | null
	subtitle?: string | null
	options?: ChartOption | null
	xAxis?: XAxis | null
	yAxis?: YAxis | null
	series:	series[]
}

interface HChartOption{
	palette ?: string | null

}

interface xAxis : {
	 title?: string | null
   categories: Array<string> | null,
   grid ?: {
	   visible?: boolean | null
   }
   label?:{
	   step?: number | null //하위 라벨 건너뛰기 레벨
   }
}

interface yAxis:{
	title?: string | null
  tick?: boolean | null
}

interface series: {
	name?: string| null
	pointLabel?: boolean | pointLabel | null // 데이터 숫자
	pointWidth?: number | null // 막대 넓이
  color?: string | null // 막대 색상
	layout?: string // 막대의 위치, ex) stack
	groupPadding?: number | null // 패딩 조절
	legendByPoint: boolean | null
  belowStyle?: {
	  fill?: string | null
    stroke?: string | null
  },
  data?: Array<number | null> | null 
  childern?: series[]
}

interface pointLabel:{
	visible?: boolean | null
	position?: string | null
	effect?: string | null
}
```

- `series:[ { childern : [ ] } ]`
    - 일 때는 series의 옵션을 모두 무시 하고, `childern` 내부에 있는 옵션들만 적용된다.
- series는 array 일수도, json일 수도 있다.
- bar에 맞게 type을 구성을 해도 chart는 동작을 한다.
- 그렇지만, typescrip 특성상 자유도가 떨어지게 되므로, chart를 구성하는데 한계는 분명 생기게 된다.
- 최소한 차트를 그리는 곳에서 에러가 나오지 않으려면, series를 그냥 any로 퉁치면 되긴 하다.
