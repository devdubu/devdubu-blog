# Real 시리즈 테스트 코드 작성
![Pasted-image-20250417175006.png](/img/이미지 창고/Pasted-image-20250417175006.png)

우선 테스트를 진행하려고 하지만, 현재 보면 `container` 에 `<div></div>` 가 정상적으로 들어간 것 같은데 인식이 안된다

왜 지?

이 문제는 realgrid에 문의를 넣고 나서 답변을 들어봐야지 알 것 같다.

## 컴포지션 분리

[https://v3-docs.vuejs-korea.org/guide/reusability/composables.html](https://v3-docs.vuejs-korea.org/guide/reusability/composables.html)

컴포지션? 컴포저블?

처음에는 헷갈렸다.

드럽게 많아 ㅡㅡ

둘의 차이점은 찾은 것 같다.

조금의 연구가 필요 해 보이지만,

:::success Vue 앱의 컨텍스트에서 **컴포저블**은 Vue 컴포지션 API를 활용하여 **상태 저장 로직**를 캡슐화하고 재사용하는 함수입니다.


:::

```tsx
// mouse.js
import { ref, onMounted, onUnmounted } from 'vue'

// 관례상, 컴포저블 함수 이름은 "use"로 시작합니다.
export function useMouse() {
  // 컴포저블로 캡슐화된 내부에서 관리되는 상태
  const x = ref(0)
  const y = ref(0)

  // 컴포저블은 시간이 지남에 따라 관리되는 상태를 업데이트할 수 있습니다.
  function update(event) {
    x.value = event.pageX
    y.value = event.pageY
  }

  // 컴포저블은 또한 이것을 사용하는 컴포넌트의 생명주기에 연결되어
  // 사이드 이펙트를 설정 및 해제할 수 있습니다.
  onMounted(() => window.addEventListener('mousemove', update))
  onUnmounted(() => window.removeEventListener('mousemove', update))

  // 관리 상태를 반환 값으로 노출
  return { x, y }
}

```

위 처럼 `onMounted` 까지 분리가 되고 적용이 되는거라면 꽤 분리를 할 때 도움이 될 것 같다.

흠 `HGrid` 자체는 분리 할게 꽤 많을 거라고 본다..

컴포저블로 해당 파일을 분리를 하고 사용자가 해당 파일을 넣었을때 올바른 로직이 작동하느냐

에 대한 테스트도 추가를 하려면 결국은 분리가 핵심이다.

사용별로 분리하고, 로직 별로 분리를 한 다음에 test를 진행해야 한다.
