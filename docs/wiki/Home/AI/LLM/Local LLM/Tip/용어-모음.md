# GGUF

**GGUF**는 '**G**eorgi **G**erganov **U**niversal **F**ormat'의 약자로, 거대 언어 모델(LLM)을 개인용 컴퓨터에서도 효율적으
로 실행하기 위해 만들어진 **파일 형식**입니다.

주요 특징은 다음과 같습니다.

- **양자화 (Quantization)**: 모델의 용량을 크게 줄이고 실행 속도를 높이기 위해 모델의 가중치(데이터의 정밀도)를 낮추는 기술입니다. 이 덕분에 고사양 서버가 아닌 일반 CPU나 GPU에서도 거대 모델을 구동할 수 있습니다.
    
- **단일 파일**: 모델의 구조, 가중치, 토크나이저 등 실행에 필요한 모든 정보가 하나의 GGUF 파일에 담겨 있어 사용이 매우 편리합니다.
    
- **CPU 중심 구동**: 기본적으로 CPU에서 모델을 실행하도록 설계되었지만, 일부 레이어를 GPU로 보내(offload) 속도를 더 높일 수도 있습니다.
    

> **쉽게 비유하자면,** GGUF는 고화질 원본 영상(원본 모델)을 화질 저하를 최소화하면서 용량을 줄여 스마트폰(개인 컴퓨터)에서도 원활하게 재생할 수 있도록 만든 '고효율 비디오 파일(mp4 등)'과 비슷합니다.

## Hardware Compatibility

:::tip 사용자의 컴퓨터 사양(특히 RAM 용량)에 맞는 최적의 모델 파일을 선택할 수 있도록 도와주는 안내표입니다.
- 양자화를 어느 정도로 적용했는지에 따라 여러 버전의 파일이 존재하며, 각 파일이 요구하는 최소 RAM 용량이 표시되어 있습니다.

:::

- **왼쪽 (2-bit, 3-bit, 4-bit...)**: **양자화 수준**을 나타냅니다. 숫자가 클수록 원본 모델에 가까워 **성능은 좋지만 파일 크기가 크고 더 많은 RAM을 필요**로 합니다. 숫자가 작을수록 압축을 많이 해서 파일이 가볍고 RAM을 적게 차지하지만, 성능이 약간 떨어질 수 있습니다.
- **오른쪽 (Q2_K, Q4_K_S, Q8_0 등)**: **구체적인 양자화 방식의 종류**입니다. 같은 비트(bit)라도 다양한 방법이 있으며, 이는 성능과 파일 크기에 미묘한 차이를 만듭니다. 일반적으로 `S` &lt; `M` &lt; `L` 순으로, 그리고 숫자가 높을수록 품질이 더 좋습니다. (예: `Q5_K_M`이 `Q4_K_S`보다 일반적으로 성능이 좋습니다.)
- **GB (기가바이트)**: 해당 모델 파일을 실행하는 데 **필요한 최소 RAM(또는 VRAM) 용량**입니다. 예를 들어, `Q4_K_S` 모델 옆에 `11.6 GB`라고 적혀 있다면, 이 모델을 사용하기 위해서는 컴퓨터에 최소 11.6GB의 여유 RAM이 있어야 한다는 의미입니다.
