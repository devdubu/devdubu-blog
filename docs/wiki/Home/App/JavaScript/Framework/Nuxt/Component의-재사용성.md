# Vue의 컴포넌트 재 사용에 대한 로직

Vue의 컴포넌트는 기본적으로 재사용에 대한 부분을 강조한다.
그리고 조금 더 나아가자면, 재사용에 대한 부분에서 문제점이 발생할 요소들이 간간히 있다.

## 공유 되는 문제

어쩌면 객체에서 많이 발생햇던 문제들이다.
객체와 일반 변수의 가장 큰 차이점은 결국 <mark>저장 방식의 차이</mark>이다.

변수를 저장하는 저장소에, 실제 값이 들어가는지 혹은 주소 값이 들어가는지에 대한 차이랄까
Vue에서도 그러한 요소들이 분명 존재한다.

우선 Nuxt3를 기준으로 설명을 하다보니, Nuxt3에서 해당 부분을 일정 공유하는 변수가 존재 한다.
보통 그걸 위해서 사용하는 변수 선언이 `useState()` 이다.

예전에는 해당 내용을 뭔 차이인지 이해를 못했는데, 이제는 조금 명확해진 것 같다.

## 프론트엔드 상태 관리

프론트 엔드에서 상태 관리에 대한 정리를 좀 하자면
vue의 관점에서 얘기를 하자면, pinia => useState => ref 순 인듯 하다.

위 함수들은 결론적으로는 반응형에 대한 변수를 의미한다.

결국 pinia는 프론트엔드 중 전반적인 상태 관리를 진행한다면, `useState` 는 해당 페이지 즉, 만약 컴포넌트 사이에서 사용했다면, 컴포넌트 간 상태 관리를 의미한다.
