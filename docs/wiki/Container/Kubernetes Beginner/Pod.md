# Pod
![Pasted-image-20240319152125.png](/img/이미지 창고/Pasted-image-20240319152125.png)
Kubernetes는 작업자 노드에 직접 컨테이너를 배포하지 않습니다.
Container는 Kubernetes라고 불리는 물체로 캡슐 형태로 포장됩니다.

:::warning Pod
- 앱의 단일 인스턴스 입니다.
-  Pod는 Kubernetes에서 만들 수 있는 가장 작은 물체 입니다.

:::

아래의 형태가 가장 간단한 형태 입니다.
![Pasted-image-20240319152634.png](/img/이미지 창고/Pasted-image-20240319152634.png)
단일 노드 Kubernetes Cluster와 응용 프로그램 단일 인스터스 포드에 축소된 단일 Docker 컨테이너에서 실행되는 것입니다.
우리의 앱에 액세스 하는 사용자 수가 증가해 앱 규모를 늘려야한다면?
로드를 공유하려면 웹 앱의 추가 인스턴스를 추가해야합니다.
:::question 추가 인스턴스를 어디서 스핀 업 할까요?
- 같은 Pod 내에서 새 컨테이너 인스턴스를 가져올까? - No
- 같은 응용 프로그램의 새로운 인스턴스로 완전히 새로운 Pod를 만드는 것입니다.

:::

보다 싶이 웹 응용 프로그램 인스턴스 2개가 동일한 Kubernetes 시스템 또는 노드에서 두 개의 분리된 포드에서 실행되고 있습니다.
:::question 유저 기반이 더 증가해서 현재 Node가 충분한 용량을 확보하지 못한다면?
- 클러스터에 새 노드에 추가 Pod를 배포할 수 있습니다.

:::

Pod는 앱에서 실행되는 컨테이너와 1대 1 관계를 갖고 있다는 겁니다.
확장하려면, 새 Pod를 만들고 축소하려면 기존 Pod는 삭제 해야 합니다.

응용프로그램 규모를 키우기 위해 기존 Pod에 <mark>추가 컨테이너</mark>를 하지 않는다.

### Multi Container PODs
Pod와 Container는 보통 1대 1로 연결되는데 한 포드에 컨테이너 하나만 들어가는 게 맞는가?
- **물론 아니다.**
![Pasted-image-20240319153742.png](/img/이미지 창고/Pasted-image-20240319153742.png)
Pod 하나에 컨테이너를 여러 개를 띄울 수 있지만, 보통은 같은 종류의 컨테이너가 여러 개 있지 않습니다.

:::warning 가끔 웹 프로그램을 지원하는 일종의 작업을 하는 도우미 컨테이너 있는 시나리오가 있을 수 있다.
- 사용자가 입력한 데이터를 처리하거나, 사용자가 업로드한 파일을 처리하는  일 등등

:::

제한적인 경우에서는 같은 포드에 해당 컨테이너 두개를 가질 수 있습니다.
:::bug 둘중 하나만 죽더라도 Pod는 죽는다.

:::

두개의 컨테이너는 localhost로 불리면서 직접 통신이 가능합니다.
같은 네트워크 공간을 공유하기 때문이다.
저장 공간 또한 쉽게 공유 가능하다.

:::warning 다중 컨테이너 Pod의 경우는 매우 드문일 이며, 왠만해서 쓰지 않는 것 이 좋다

:::
