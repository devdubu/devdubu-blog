# EC2 Instance Store 
- EBS 볼륨은 network driver로 성능 자체는 뛰어나지만 제한된 퍼포먼스를 가지고 있습니다.
	- 이보다 더 높은 성능을 요구할 때도 있고, 이때는 EC2 인스턴스에 연결된 하드웨어 디스크 성능이 향상 되어야합니다.
- EC2 인스턴스 머신은 가상머신이지만, 실제로는 하드웨어 서버에 연결되어 있습니다.
	- 해당 서버에 물리적으로 연결된 디스크 공간을 갖습니다.
	- 따라서 특정 유형의 EC2 인스턴스는 EC2 인스턴스 스토어라고 불리며, 이는 해당하는 물리적 서버에 연결된 하드웨어 드라이를 가르킵니다.
## 사용 이유
- I/O 성능 향상을 위해 활용할 수 있습니다.
- 이들이 훌륭한 처리량을 갖추고 있어서 매우 향상된 디스크 성능을 요할 때 활용할 수 있도록 확보할 필요가 있습니다.
- 이때 주의해야할 점은 EC2 인스턴스, 즉 인스턴스 스토어를 중지 또는 종료하면 해당 스토리지 또한 손실된다는 점입니다.
	- 이와 같이 이를 임시 스토리지라고도 불립니다.
- 이를 통해서 EC2 인스턴스 스토어가 장기적으로 데이터를 보관할 만한 장소가 될 수 없음을 대변하고 습니다.
- 버퍼, 캐시, 스크래치 데이터 또는 임시 컨텐츠를 사용하려는 경우 이들을 보관할 좋은 장소가 되지만, 장기적인 스토리지는 될 수 없습니다.
	- 장기적인 스토리지는 EBS가 적합합니다.
- EC2 인스턴스의 기본 서버에 장애가 발생할 시에는 인스턴스와 연결된 인스턴스 스토어에 장애가 발생하므로 데이터 손실에 대한 위험이 존재합니다.
- 따라서 EC2 인스턴스 스토어를 사용할 때에 데이터를 백업해 두거나 혹은 복제해 둬야 합니다.
