# Amazon S3 (Simple Storage Service)

## 1. Overview
:::tip Amazon S3란?
- **객체 스토리지(Object Storage)** 서비스입니다.
- 용량 무제한 (단일 파일 최대 5TB).
- 99.999999999% (11 9s)의 내구성을 제공합니다 (여러 가용 영역에 중복 저장).
- HTTP/HTTPS API를 통해 데이터에 접근합니다.
:::
![Pasted-image-20231023102629.png](/img/Pasted-image-20231023102629.png)

### Key Features
- **버킷(Bucket)**: 객체를 저장하는 컨테이너 (리전 단위, 이름은 전역적으로 고유해야 함).
- **객체(Object)**: 파일 데이터 + 메타데이터 + 키(Key).
- **버전 관리(Versioning)**: 파일의 여러 버전을 유지하여 실수로 삭제/덮어쓰기 방지.
- **보안**: 저장 데이터 암호화(SSE), 전송 중 암호화(SSL/TLS), 버킷 정책 및 ACL로 접근 제어.
- **정적 웹 사이트 호스팅**: S3 버킷을 웹 서버처럼 사용하여 HTML/JS/CSS 파일을 서비스할 수 있습니다.

## 2. Storage Classes
용도와 비용에 따라 다양한 스토리지 클래스를 제공합니다.

![Pasted-image-20231023102946.png](/img/Pasted-image-20231023102946.png)

| 클래스 | 설명 | 용도 |
| -- | -- | -- |
| **S3 Standard** | 기본값. 높은 내구성, 가용성, 성능. | 자주 액세스하는 데이터, 클라우드 앱, 정적 웹사이트. |
| **S3 Standard-IA** | Infrequent Access. 저장 비용은 낮으나 액세스 비용 발생. | 자주는 아니지만 필요할 때 빠르게 접근해야 하는 데이터. 백업. |
| **S3 One Zone-IA** | 단일 AZ에만 저장 (내구성 낮음). | 다시 생성 가능한 데이터, 보조 백업. |
| **S3 Glacier** | 장기 보관용 아카이브. | 몇 분~몇 시간의 검색 시간 허용. |
| **S3 Glacier Deep Archive** | 가장 저렴함. | 12시간 이상의 검색 시간 허용 (규제 준수용 장기 보관). |
| **S3 Intelligent-Tiering** | 액세스 패턴을 모니터링하여 자동으로 계층 이동. | 액세스 패턴을 알 수 없는 데이터. |
