# Ansible?
앤서블은 마이클 데한에 의해 Ansible Works 라는 이름으로 14년도에 처음 소개 되었다.
그 후 레드헷이 인수하여 개발 관리 하고 있습니다.

앤서블을 오픈소스로 개발한 마이클은 시스템 관리 소프트웨어를 작성하는 일을 주로 햇습니다.

## No Agentless
기존 자동화 도구인 Puppet 이나 Chef는 자동화 대상 서버에 별도의 에이전트를 설치하고, 이를 통해 자동화 업무를 수행합니다.
이러한 데몬 형식의 에이전트에 기반한 자동화 도구는 관리를 위한 복잡한 추가 작업이나 운영체제 버전에 따라 추가 패키지나 모듈을 설치하는 등의 작업이 발생합니다.

## 멱등성
:::tip 멱등성?
:::

 >- `idempotent` 은 동일한 연산을 여러 번 적용하더라도 결과가 달라지지 않는 성질
 >- 멱등성 법칙의 개념은 추상대수학과 함수형 프로그래밍의 여러 부분에서 사용 중 입니다.

앤서블은 이러한 멱등성과 함께 시스템을 원하는 상태로 표현하여 유지하도록 설계되어 있어, 동일한 운영 작업을 여러 번 실행해도 같은 결과를 냅니다.

## 쉬운 사용법과 다양한 모듈 제공
앤서블은 다른 자동화 도구에 비해 복잡하지 않아, 자동화 절차 과정을 이해하기 쉽습니다.
자동화 단계는 에디터만 있다면, YAML 문법을 사용하여, 쉽게 작성 가능하고 읽을 수 있습니다.
또한 파일 복사와 같은 일반 시스템 관리 모듈 부터 다양한 환경의 퍼블릭 클라우드 관련 모듈및 컬렉션까지 제공하므로, 쉽게 플레이북 예제를 찾아보고 자동화를 수행할 수 있습니다.

# Ansible 사용처
앤서블은 주로 시스템 환경을 설치하거나, 애플리케이션을 설치할 대, 퍼블릭이나 혹은 프라이빗 클라우드 시스템의 가상 서버 등을 생성할 때, 혹은 매일 점검해야하는 시스템의 상태를 체크할 때도 사용 가능합니다.

뿐만 아니라, 특정 시스템을 구축할 때도 앤서블을 사용하는 것을 종종 볼 수 있습니다.
따라서 사용 빈도가 많고 여러 시스템에 동일한 작업을 해야할 상황에 앤서블을 주로 사용합니다.

## Ansible의 어디에 쓰이는가

앤서블은 리눅스, MacOS BSD 계열 유닉스, WSL 을 지원하는 윈도우에 파이썬과 앤서블 코어만 설치하면 어디에서나 플레이북을 잓헝하고 이를 실행시킬 수 있습니다.

앤서블은 이렇게 앤서블 코어가 설치되고 플레이북을 작성하여 실행할 수 있는 <mark>제어노드(Control Node)</mark> 와 플레이북이 실행되어 애프리케이션 설치나 클라우드 시스템의 가상 서버 생성과 같은 작업이 수행되는 <mark>관리 노드(Managed Node)</mark>로 구성됩니다.
