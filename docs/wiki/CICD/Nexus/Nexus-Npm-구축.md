# Role 생성
![Screenshot-2025-03-15-at-11.22.31-PM.png](/img/이미지 창고/Screenshot-2025-03-15-at-11.22.31-PM.png)

생성한 repo에 접근하기 위해서는 role을 생성하고 계정에 등록해주어야 합니다.

repo에 대한 권한은 2가지로 나뉩니다.

- nx-repository-admin-\{저장소 종류}-\{저장소 이름}-\{작업}: 해당 저장소에 대한 설정
- nx-repository-view-\{저장소 종류}-\{저장소 이름}-\{작업}: 해당 저장소에 대한 사용

admin 권한이 있으면 상단 톱니바퀴 버튼에서 설정을 관리할 수 있고, view 권한이 있으면 톱니바퀴 좌측 repo를 시각적으로 확인할 수 있습니다.

위에서 마지막에 해당되는 5가지 작업 내용은 다음과 같습니다.

1. **add (추가)**
    - **기능**: 이 권한은 사용자가 새로운 아티팩트나 컨텐츠를 Nexus 저장소에 추가할 수 있게 합니다.
    - **적용**: 개발자나 배포 팀이 새로운 빌드나 라이브러리를 저장소에 업로드할 때 필요합니다.
2. **browse (탐색)**
    - **기능**: Browse 권한은 사용자가 저장소의 내용을 탐색하고, 아티팩트의 목록을 볼 수 있게 합니다. 하지만, 아티팩트의 실제 내용을 읽거나 다운로드할 수 있는 권한은 포함하지 않습니다.
    - **적용**: 일반 사용자나 외부 이해관계자가 저장소의 내용을 확인할 필요가 있을 때 유용합니다.
3. **delete (삭제)**
    - **기능**: 이 권한을 가진 사용자는 저장소에서 아티팩트나 컨텐츠를 삭제할 수 있습니다.
    - **적용**: 관리자나 특정 권한을 가진 사용자가 오래된 또는 불필요한 아티팩트를 정리할 때 사용됩니다.
4. **edit (편집)**
    - **기능**: Edit 권한은 사용자가 저장소의 메타데이터나 구성 설정을 변경할 수 있게 합니다. 이는 아티팩트 자체의 수정이 아니라 저장소 설정의 수정을 의미합니다.
    - **적용**: 저장소 관리자나 DevOps 팀이 저장소 설정을 최적화하거나 변경해야 할 때 필요합니다.
5. **read (읽기)**
    - **기능**: Read 권한은 아티팩트의 실제 내용을 읽고 다운로드할 수 있는 권한을 제공합니다.
    - **적용**: 대부분의 개발자와 사용자에게 필요한 기본 권한으로, 아티팩트의 의존성 해결이나 빌드 프로세스에서 중요합니다.

## Realms 설정



Realms에 `npm Bearer Token Realm`을 Active로 설정합니다.

이걸 설정하지 않으면 npm login 후 클라이언트 ↔ nexus 간 토큰 방식의 인증이 되지 않아 E401 에러가 발생하게 됩니다.
