# 컴퓨터가 이해하는 정보
컴퓨터는 0과 1로 표현된 정보만 이해합니다.
정보에는 두 종류가 존재 합니다.

바로 <mark>데이터</mark>와 <mark>명령어</mark> 입니다.

![Diagram12.svg](/img/이미지 창고/Diagram12.svg)

:::info 데이터
- 컴퓨터가 이해하는 숫자, 문자, 이미지, 동영상과 같은 정적인 정보를 말합니다.

:::

컴퓨터와 주고 받는 정보나 컴퓨터에 저장된 정보를 가리킬 때 편하게 데이터를 통칭하기도 합니다.

:::info 명령어
- 컴퓨터를 실질적으로 작동 시키는 더 중요한 정보는 명령어 입니다.
- 데이터는 명령어 없이는 아무것도 할 수 없는 정보 덩어리 입니다.

:::

# 컴퓨터의 4가지 핵심 부품


:::note 주기억장치의 종류
- RAM(Random Access Memory) / ROM(Read Only Memory)
- 메모리 라는 용어는 보통 RAM을 지칭합니다.

:::

## 메모리
컴퓨터가 이해하는 정보는 명령어와 데이터 입니다.
메모리는 현재 실행되려면 반드시 메모리에 저장되어야 합니다.
이 때 컴퓨터가 빠르게 작동하기 위해서는 메모리 속 명령어와 데이터가 중구난방으로 저장되어 있으면 안됩니다.
![Pasted-image-20240804152132.png](/img/이미지 창고/Pasted-image-20240804152132.png)
메모리에 저장된 값에 빠르고 효율적으로 접근하기 위해서 <mark>주소</mark> 라는 개념이 사용됩니다.
현실에서는 우리가 주소로 원하는 위치를 찾아 갈 수 있듯이 컴퓨터에서도 주소로 메모리 내 원하는 위치에 접근 할 수 있습니다.

물론 이것은 메모리를 이해하기 쉽게 그린 것뿐이다.
명령어와 데이터는 모두 0과 1로 표현되기 때문에, 겉보기에는 마치 0과 1로 이루어진 데이터를 저장한 것 처럼 보입니다.

## CPU
CPU는 컴퓨터의 두뇌 입니다.
CPU는 메모리에 저장된 명령어를 읽어 들이고, 읽어 들인 명령어를 해석하고 실행하는 부품입니다.
![Pasted-image-20240804152427.png](/img/이미지 창고/Pasted-image-20240804152427.png)CPU의 역할과 작동 원리를 구체적으로 이해하기 위해서는 CPU 내부 구성 요소를 알아야 합니다.
이 책에서 학습할 CPU 내부 구성 요소 중 가장 중요한 세 가지 **산술논리연산장치**, **레지스터**, **제어 장치** 입니다.

### ALU
- 쉽게 말하면 계산기 이다.
- 계산 만을 위해 존재하는 부품이다.
- 컴퓨터 내부에서 수행되는 대부분의 계산은 ALU가 도 맡아서 수행한다.
### 레지스터
- CPU 내부의 작은 임시 저장 장치이다.
- 프로그램을 실행하는 데 필요한 값들을 임시로 저장합니다.
- CPU 안에는 여러 개의 레지스터가 존재하고, 각기 다른 이름과 역할을 가지고 있습니다.
### 제어장치
- 제어 신호 라는 전기 신호를 내보내고 명령어를 해석하는 장치입니다.
- 여기서 제어 신호란?
	- 컴퓨터 부품들을 관리하고 작동 시키기 위한 일종의 전기 신호 입니다.

> CPU가 메모리에 저장된 값을 읽고 싶을 땐 메모리를 향해 **메모리 읽기** 라는 제어 신호를 보낸다.
> CPU가 메모리에 어떤 값을 저장하고 싶을 땐 메모리를 향해 **메모리 쓰기**라는 제어 신호를 보낸다.

#### 제어장치 작동 원리

![Pasted-image-20240804153036.png](/img/이미지 창고/Pasted-image-20240804153036.png)
1. 제어 장치는 1번지에 저장된 명령어를 읽어 들이기 위해 메모리에 메모리 읽기 제어 신호를 보냅니다.
![Pasted-image-20240804153147.png](/img/이미지 창고/Pasted-image-20240804153147.png)
2. 메모리는 1번지에 저장된 명령어를 CPU에 건네주고, 이 명령어는 레지스터에 저장됩니다
	1. 이 명령어는 레지스터에 저장됩니다.
	2. 제어 장치는 읽어들인 명령어를 해석한 뒤 3번지와 4번지에 저장된 데이터가 필요하다고 판단
	3. 제어 장치는 3번지와 4번지에 저장된 데이터를 읽어 들이기 위해 메모리에 **메모리 읽기** 제어 신호를 보냅니다.

![Pasted-image-20240804153338.png](/img/이미지 창고/Pasted-image-20240804153338.png)
3. 메모리는 3번지와 4번지에 저장된 데이터를 CPU에 건네주고, 이 데이터 들은 서로 다른 레지스터에 저장 됩니다.
	1. ALU는 읽어들인 데이터로 연산을 수행합니다.
	2. 계산의 결괏값은 레지스터에 저장됩니다.
	3. 계산이 끝났다면, 첫 번째 명령의 실행은 끝이 납니다.
![Pasted-image-20240804153551.png](/img/이미지 창고/Pasted-image-20240804153551.png)
4. 제어 장치는 2번지에 저장된 다음 명령어를 읽어 들이기 위해 메모리에 **메모리 읽기** 제어 신호를 보냅니다.
![Pasted-image-20240804153638.png](/img/이미지 창고/Pasted-image-20240804153638.png)
5. 메모리는 2번지에 저장된 명령어를 CPU에 건네주고, 이 명령어는 레지스터에 저장됩니다.
	1. 제어 장치는 이 명령어를 해석한 뒤, 메모리에 계산 결과를 저장해야 한다고 판단
	2. 제어 장치는 계산 결과를 저장하기 위해 메모리에 **메모리 쓰기** 제어 신호와 함께 계산 결과인 220을 보냅니다.
	3. 메모리가 계산 결과를 저장하면 두 번째 명령어의 실행도 끝납니다.
![Pasted-image-20240804154036.png](/img/이미지 창고/Pasted-image-20240804154036.png)

## 보조 기억  장치
앞서 메모리는 실행되는 프로그램의 명령어와 데이터를 저장한다고 했지만, 이 메모리는 두 가지 치명적인 약점이 있습니다.
첫째는 가격이 비싸 저장 용량이 적다는 점, 둘재는 전원이 꺼지면 저장된 내용을 읽는 다는 점이다.
컴퓨터로 작업하는 도중에 전원이 꺼져서 작업한 내역을 잃어본적 있나요?
전원이 꺼지면 작업한 내역을 잃게 되는 이유는 실행중인 프로그램들은 메모리에 저장되는데 메모리는 전원이 꺼지면 저장된 내용이 날아가기 때문이다.

:::note 보조 기억 장치?
- 메모리 보다 크기가 크고 전원이 꺼저도 저장된 내용을 잃지 않는 메모리를 보조할 저장장치가 필요하게 되었는데, 이 저장 장치가 보조 기억 장치이다.

:::

![Pasted-image-20240804154413.png](/img/이미지 창고/Pasted-image-20240804154413.png)

하드 디스크 , SSD, USB 메모리, DVD, CD-ROM과 같은 저장 장치가 보조 기억 장치의 일종입니다.
컴퓨터 전원이 꺼져도 컴퓨터에 파일이 남아 있었던 이유는 우리가 파일을 보조 기억 장치에 저장했기 때문입니다.
메모리가 현재 실행되는 프로그램을 저장한다면, 보조 기억 장치는 보관할 프로그램을 저장한다고 생각해도 좋습니다.

### 입출력 장치
![Pasted-image-20240804214219.png](/img/이미지 창고/Pasted-image-20240804214219.png)
마이크, 스피커, 프린터, 마우스, 키보드 처럼 컴퓨터 외부에 연결되어 컴퓨터 내부와 정보를 교환하는 장치를 의미한다.

### 메인 보드와 시스템 버스
![Pasted-image-20240804214321.png](/img/이미지 창고/Pasted-image-20240804214321.png)
지금 까지 설명한 컴퓨터의 핵심 부품은 모두 메인 보드 라는 판에 연결됩니다.
메인 보드는 마더보드라고 부르기도 합니다.

메인 보드에 연결된 부품들은 서로 정보를 주고 받을 수 있는데, 이는 메인 보드 내부에 버스 라는 통로가 있기 때문입니다.
컴퓨터 내붑에는 다양한 종류의 통로, 즉 버스가 있습니다.
하지만 여러 버스 가운데 컴퓨터 네 가지 핵심 부품을 연결하는 가장중요한 버스는 시스템 버스 입니다.

시스템 주소 버스는 주로 버스, 데이터 버스, 제어 버스로 구분되어 있습니다.

#### 주소 버스
- 주소를 주고 받는 통로
#### 데이터 버스
- 명령어와 데이터를 주고 받는 통로
#### 제어 버스

- 제어 신호를 주고 받는 통로

시스템 버스를 조금 더 자세히 이해하기 위해 앞서 설명한 CPU의 작동 예시를 다시 본다면,
CPU가 메모리 속 명령어를 읽어 들이기 위해 제어 장치에서 **메모리 읽기** 라는 신호를 내보낸다고 했습니다.
그런데 사실 CPU가 메모리를 읽을 땐 제어 신호만 내보내지 않습니다.
실제로 제어 버스로 메모리 읽기 제어 신호를 내보내고, 주소 버스로 읽고자 하는 주소를 내보냅니다.
그러면 메모리는 데이터 버스로 CPU가 요청한 주소에 있는 내용을 보냅니다.

![Pasted-image-20240804214731.png](/img/이미지 창고/Pasted-image-20240804214731.png)

![Diagram-5.svg](/img/이미지 창고/Diagram-5.svg)

그리고 메모리에 어떤 값을 저장할 때도 CPU는 데이터 버스를 통해 메모리에 저장할 값을, 주소 버스를 통해 저장할 주소를 제어 버스를 통해 메모리 쓰기 제어 신호를 내보냅니다.
